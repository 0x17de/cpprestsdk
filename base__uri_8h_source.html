130  /// <summary>
131  /// A flexible, protocol independent URI implementation.
132  ///
133  /// URI instances are immutable. Querying the various fields on an emtpy URI will return empty strings. Querying
134  /// various diagnostic members on an empty URI will return false.
135  /// </summary>
136  /// <remarks>
137  /// This implementation accepts both URIs ('http://msn.com/path') and URI relative-references
138  /// ('/path?query#frag').
139  ///
140  /// This implementation does not provide any scheme-specific handling -- an example of this
141  /// would be the following: 'http://path1/path'. This is a valid URI, but it's not a valid
142  /// http-uri -- that is, it's syntactically correct but does not conform to the requirements
143  /// of the http scheme (http requires a host).
144  /// We could provide this by allowing a pluggable 'scheme' policy-class, which would provide
145  /// extra capability for validating and canonicalizing a URI according to scheme, and would
146  /// introduce a layer of type-safety for URIs of differing schemes, and thus differing semantics.
147  ///
148  /// One issue with implementing a scheme-independent URI facility is that of comparing for equality.
149  /// For instance, these URIs are considered equal 'http://msn.com', 'http://msn.com:80'. That is --
150  /// the 'default' port can be either omitted or explicit. Since we don't have a way to map a scheme
151  /// to it's default port, we don't have a way to know these are equal. This is just one of a class of
152  /// issues with regard to scheme-specific behavior.
153  /// </remarks>

347  /// <summary>
348  /// A loopback URI is one which refers to a hostname or ip address with meaning only on the local machine.
349  /// </summary>
350  /// <remarks>
351  /// Examples include "locahost", or ip addresses in the loopback range (127.0.0.0/24).
352  /// </remarks>
353  /// <returns><c>true</c> if this URI references the local host, <c>false</c> otherwise.</returns>
354  bool is_host_loopback() const
355  {
356  return !is_empty() && ((host() == _XPLATSTR("localhost")) || (host().size() > 4 && host().substr(0,4) == _XPLATSTR("127.")));
357  }

361  /// </summary>

370  /// <summary>
371  /// A portable URI is one with a hostname that can be resolved globally (used from another machine).
372  /// </summary>
373  /// <returns><c>true</c> if this URI can be resolved globally (used from another machine), <c>false</c> otherwise.</returns>
374  /// <remarks>
375  /// The hostname "localhost" is a reserved name that is guaranteed to resolve to the local machine,
376  /// and cannot be used for inter-machine communication. Likewise the hostnames "*" and "+" on Windows
377  /// represent wildcards, and do not map to a resolvable address.
378  /// </remarks>
379  bool is_host_portable() const
380  {
381  return !(is_empty() || is_host_loopback() || is_host_wildcard());
382  }
